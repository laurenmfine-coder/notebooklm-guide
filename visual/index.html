Essential Prompts for Visual Learners

Comparison Table

Create a detailed comparison table for [list 3-5 related conditions/medications]. Include columns for: mechanism, presentation, key diagnostic findings, treatment, and important side effects/complications.

Algorithm Description

Generate a step-by-step algorithm for managing [condition/symptom]. Start from initial presentation and include decision points, diagnostic steps, and treatment pathways. Describe it clearly so I can draw it as a flowchart.

Visual Framework

Describe a visual framework for understanding [topic]. Organize it hierarchically or in a logical flow that I can draw or diagram. Include all key components and how they relate.

Concept Map Description

Create a concept map showing how [topic] connects to related concepts. Describe the central idea and all branching concepts with their relationships.

Common Questions

Can NotebookLM create actual diagrams?
NotebookLM generates text descriptions, not images. However, it can describe diagrams in enough detail that you can draw them yourself or use other tools to create them. The act of drawing often helps visual learners retain information better anyway.
How do I turn tables into visual aids?
Copy the table into a spreadsheet, then use color-coding and formatting. Or redraw the information as a diagram, flowchart, or mind map‚Äîthe reprocessing helps with retention.
What's the best way to request algorithm descriptions?
Be specific about what you want to visualize. Ask for "decision points," "branches," or "pathways" and specify you want to draw it as a flowchart. This helps NotebookLM structure the response visually.
Can I request specific table columns?
Yes! Be explicit about what columns you want. For example: "Include columns for mechanism, indications, side effects, and contraindications." This gives you exactly the comparison you need.
